The Problem

Three fractures in the pipeline

01
Idle
Design is done. Then we wait — days — for dev to build what we already see in Figma.
02
Drift
Design intent lost in translation. Even with Dev Mode, back-and-forth never ends.
03
Loop
Edge cases surface after dev is done. Fix design, wait for rebuild, repeat.

The Shift

What if designers could deploy themselves?
Figma Make → Code → Deploy

Skip the handoff. Go from design to a live build — directly.

1

Challenge

Output Fidelity

Figma Make generates code, but does it match the design? Not out of the box. Output quality depends entirely on the constraints you feed it.

design-guidelines.md global.css daily iteration

We tested different guideline configurations every day to find what makes AI output stable.

2

Challenge

Structural Integrity

Looking right is surface-level. The code needs to be structured, stable, and usable by frontend and backend engineers — not just a visual demo.

As designers who don't write code, our only lever was the prompt. We experimented relentlessly to make the output architecturally sound.

prompt engineering code structure trial and error
3

Challenge

Beyond a Pretty Page

Everyone asked: "How is this different from a Figma prototype?"

The answer: data separation. We extract a data.ts from the generated code. Backend engineers then derive a data schema from it. The result is a real frontend that runs on real data — not a static mock.

data.ts extraction data schema real API integration

The Result

Dexco

A deployment platform built for designers. From Figma Make to a live, data-connected build — without writing code.
